# Contributing to Budget App

Thank you for your interest in contributing to Budget App! This document explains how you can report bugs, request features, and get help.

## ğŸ› Reporting Bugs

If you've found a bug, please help us fix it by reporting it:

1. **Check existing issues** - Search [existing issues](https://github.com/codescribblr/budget-app/issues) to see if someone has already reported it
2. **Create a new issue** - If it hasn't been reported, [create a bug report](https://github.com/codescribblr/budget-app/issues/new?template=bug_report.md)
3. **Provide details** - Include:
   - Clear description of the bug
   - Steps to reproduce
   - Expected vs actual behavior
   - Screenshots if applicable
   - Browser/device information

## ğŸ’¡ Requesting Features

Have an idea for a new feature or improvement?

1. **Check existing requests** - Search [existing issues](https://github.com/codescribblr/budget-app/issues?q=label%3Aenhancement) to see if it's already been suggested
2. **Create a feature request** - If it's new, [submit a feature request](https://github.com/codescribblr/budget-app/issues/new?template=feature_request.md)
3. **Describe your idea** - Include:
   - What problem it solves
   - How you envision it working
   - Why it would be valuable
   - Any examples or mockups

Need help using the app or have questions?

- **GitHub Discussions** - Ask questions in [Discussions](https://github.com/codescribblr/budget-app/discussions)
- **Documentation** - Check the in-app help center (click "Help" in the navigation) for guides and tutorials
- **Search Issues** - Someone may have already asked your question

## ğŸ¯ Issue Labels

We use labels to organize issues:

- `bug` - Something isn't working correctly
- `enhancement` - New feature or improvement request
- `documentation` - Documentation improvements
- `good first issue` - Good for newcomers
- `help wanted` - Extra attention needed
- `question` - Further information requested
- `wontfix` - This will not be worked on

## ğŸ“‹ Issue Lifecycle

1. **New** - Issue is created and awaiting review
2. **Triaged** - Issue has been reviewed and labeled
3. **In Progress** - Work has started on the issue
4. **Completed** - Issue has been resolved and closed
